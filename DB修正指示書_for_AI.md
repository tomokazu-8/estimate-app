# 見積システム DBファイル修正 — チャット AI 向け指示書

この指示書と一緒に `data/material_db.json`・`data/bukariki_db.json` を渡してください。
AI はこの説明を読んで DB の内容を正しく修正してください。

---

## 1. ファイル概要

| ファイル | 役割 | 件数目安 |
|---|---|---|
| `data/material_db.json` | 材料の品名・規格・単価を格納した材料マスタ | 約540件 |
| `data/bukariki_db.json` | 材料の取付人工数（歩掛）を格納した歩掛マスタ | 約100件 |

## 2. material_db.json — フィールド定義

```json
{
  "n": "電力用ｹｰﾌﾞﾙ 600V VV-F",   // 品名（検索・選択時に行の「名称」に入る）
  "s": "VV-F 1.6 - 2C",            // 規格（検索・選択時に行の「規格」に入る）
  "u": "ｍ",                         // 単位（m / 個 / 台 / 式 / 組 / 面 / 本 など）
  "c": "cable",                      // カテゴリ（後述）
  "ep": 108.0,                       // 見積単価（円）← 検索結果に表示・明細行に入力される
  "r": 0.754,                        // 原価率（0〜1の小数）
  "cp": 72.0,                        // 原価（ep × r に近い値）
  "a": 8                             // 使用頻度スコア（過去実績で使った回数。大きいほど上位表示）
}
```

### カテゴリ一覧（`c` フィールド）

| 値 | 意味 | 件数 |
|---|---|---|
| `cable` | 電線・ケーブル類 | 33 |
| `conduit` | 電線管・管付属品 | 42 |
| `box` | ボックス類 | 23 |
| `device` | 配線器具（コンセント・スイッチ等） | 77 |
| `fixture` | 照明器具・弱電機器・その他 | 337 |
| `panel` | 分電盤・開閉器 | 6 |
| `fire` | 火災感知器・火報機器 | 7 |
| `ground` | 接地工事材 | 3 |
| `dimmer` | 調光器 | 10 |
| `accessories` | 付属品 | 2 |

### 検索ロジック（アプリ内の動作）

1. キーワードをスペースで分割し、**すべての語が `n + " " + s` に含まれる場合にマッチ**
2. カテゴリフィルタが選択されていれば、その `c` 値でさらに絞り込む
3. 最大50件表示（配列の先頭から取得するため、`a` 値が高い＝よく使う品目を配列の前に置くこと）
4. ユーザーが選択すると `n`, `s`, `u`, `ep` が明細行に転記される

### 文字コード・表記ルール

- 品名・規格には**カタカナは半角カナ**（ｹｰﾌﾞﾙ・ｺﾝｾﾝﾄ など）を使う
- 漢字・ひらがなはそのまま（電力用・合成樹脂製 など）
- 型番・サイズ表記はアルファベット大文字 + スペース区切り（`VV-F 1.6 - 2C`）
- 電線断面積の `mm²` は `°` で代用している（例: `CV 14°`）

## 3. bukariki_db.json — フィールド定義

```json
{
  "n": "電力用ｹｰﾌﾞﾙ 600V VV-F",   // 品名キーワード（前方一致・部分一致で検索される）
  "s": "VV-F 1.6 - 2C",            // 規格キーワード（空文字 "" の場合はどの規格にもマッチ）
  "b": 0.013,                       // 歩掛値（人工/数量単位）
  "c": "cable",                     // カテゴリ（参考情報、検索には使用しない）
  "u": "ｍ"                          // 単位（参考情報）
}
```

### 歩掛マッチングロジック（アプリ内の動作）

明細行の `名称 + 規格` 文字列に対して以下の順で歩掛を探す：

1. **完全一致パス**：`n` が含まれ、かつ `s != ""` なら `s` も含まれる → そのレコードの `b` を採用
2. **近似一致パス**：`n` の最初の語（3文字以上）が含まれる → そのレコードの `b` を採用
3. **カテゴリデフォルト**：以下のキーワードが含まれれば固定値を返す

| キーワード | デフォルト歩掛 |
|---|---|
| ｹｰﾌﾞﾙ / vv-f / cv / cvt / iv / 同軸 / cpev | 0.01 |
| 電線管 / pf- / ve / fep / ﾈｼﾞﾅｼ | 0.02 |
| ｺﾝｾﾝﾄ / ｽｲｯﾁ / ﾌﾟﾚｰﾄ | 0.05 |
| 分電盤 / 開閉器 | 0.25 |
| 火災 / 感知 / 報知 | 0.08 |
| 接地 | 0.15 |

4. **最終デフォルト**：0.07（照明器具扱い）

### 歩掛の目安（人工/個または人工/m）

| 品種 | 目安 |
|---|---|
| VVF 1.6-2C（m単位） | 0.013〜0.017 |
| PF管 16mm（m単位） | 0.031 |
| 埋込コンセント1個 | 0.067 |
| 埋込スイッチ片切1個 | 0.054 |
| ダウンライト1台 | 0.07〜0.13 |
| 分電盤1面 | 2.5〜5.0 |
| 感知器1個 | 0.08 |

## 4. よくある修正パターン

### パターンA：品名が検索でヒットしない

**原因**：`n` または `s` の表記が検索キーワードと一致しない
**対処**：`n` / `s` の表記を実際に検索されるキーワードに合わせる

```json
// 修正前（ヒットしない）
{"n": "600V VVF ケーブル", "s": "2×1.6"}

// 修正後（「VV-F 1.6」で検索してヒットするように）
{"n": "電力用ｹｰﾌﾞﾙ 600V VV-F", "s": "VV-F 1.6 - 2C"}
```

### パターンB：単価が実勢価格と乖離している

**対処**：`ep`（見積単価）と `cp`（原価）を正しい値に更新。`r = cp/ep` になるように `r` も更新する。

```json
// 修正例：単価を更新
{"n": "...", "s": "...", "ep": 150.0, "cp": 113.0, "r": 0.753, ...}
```

### パターンC：品目が存在しない（新規追加）

- 既存エントリを参考にして同じフィールドを持つオブジェクトを配列に追加する
- `a` は初期値として `1` を設定する
- よく使う品目は配列の**前の方**に追加すると上位表示される

### パターンD：歩掛の値が実態と合わない

`bukariki_db.json` の対象レコードの `b` 値を修正する。
品目が存在しない場合は同カテゴリの近い品目を参考に新規追加する。

---

## 5. 修正時の注意事項

1. **JSONフォーマットを壊さない**こと（末尾カンマ禁止、文字列は必ずダブルクォート）
2. `c` フィールドは必ず上記カテゴリ一覧の値を使う
3. `ep`・`cp` は数値型（文字列にしない）：`"ep": 108.0` ○、`"ep": "108"` ×
4. `r` は `cp / ep` の小数で小数点3桁まで
5. 文字種は既存エントリに合わせ、**カタカナは半角カナを使う**
6. `s`（規格）が特に区別不要な場合は空文字 `""` にする
